# Phase 08 Plan: Enhance Heuristics & Analysis

<objective>
Enhance the heuristic analysis engine with custom URL unshortening, local PhishTank DB lookups, and Display Name mismatch spoofing detection.
</objective>

<must_haves>
- D-01: Must follow URL redirects manually via `requests` without exceeding a maximum hop limit (e.g., 5 hops) to prevent infinite loops.
- D-02: Must query a local `phishtank_urls` SQLite table for domain checks instead of calling external APIs per request.
- D-03: Must extract display names using `email.utils.parseaddr` and check for domain mismatches against high-profile brands.
</must_haves>

## Tasks

<task>
  <id>08-01-01</id>
  <action>
    Modify `analysis/heuristics.py`:
    1. Add a helper method `_unshorten_url(url: str) -> str` that uses `requests.Session` with a max redirect limit of 5. It should catch network exceptions and return the original URL if it fails.
    2. Add a helper method `_check_phishtank(url: str) -> bool` that queries the `phishguard.db` database for the presence of the URL in the `phishtank_urls` table.
